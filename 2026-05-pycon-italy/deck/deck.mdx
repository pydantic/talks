# An agent

Structured, type-safe output with Pydantic AI.

```py
from datetime import datetime
from pydantic import BaseModel
from pydantic_ai import Agent

class ConferenceTalk(BaseModel):
    title: str
    start_time: datetime
    room: str | None = None

conference_talk_agent = Agent(
    'gateway/anthropic:claude-haiku-4-5',
    output_type=list[ConferenceTalk],
    instructions='Extract information about conference talks',
)
```

# Make the agent durable

One line (and an import) to make the agent durable.

```py
...

from pydantic_ai.durable_exec.temporal import TemporalAgent

durable_conference_talk_agent = TemporalAgent(conference_talk_agent)
```

**No need to manually wrap your tool calls as activities.**

</Slide>

<Slide tab="durable">

# How this works

- The **agent loop** lives in the workflow - completely deterministic
- **Every external call** becomes an activity:
  - model requests
  - `@agent.tool` functions
  - MCP server calls
- One `TemporalAgent` wraps your agent and wires it all up

</Slide>

<Slide tab="durable">

# What this gets you

- **Resume mid-run** - agents resume after a worker dies - from where it left off
- **Automatic retries** - flaky tools, rate-limited models, etc
- **Long-running by default** - `workflow.sleep(timedelta(days=7))` just works
- **No bespoke orchestration** - no queues, no checkpoint tables, no state machines you have to write yourself

</Slide>

## 20 Questions
# <span style={{ color: 'var(--accent-secondary)' }}>LLM vs LLM</span>

<div style={{ display: 'flex', gap: '3rem', marginTop: '0.5rem', alignItems: 'flex-start' }}>
<div style={{ flex: '0 0 55%' }}>

**Rules:**

1. Player 1 thinks of a secret object, person, or place
2. Player 2 tries to guess the object by asking yes/no questions
3. Each question helps Player 2 narrow down the potential options
4. If Player 2 guesses correctly within 20 questions, Player 2 wins

</div>
<div style={{ flex: 1, background: 'rgba(229,32,233,0.08)', border: '1px solid var(--accent)', borderRadius: '10px', padding: '1.2rem', marginTop: '-2.5rem' }}>

**Example:**

Player 1 thinks of **"Elephant"**

Player 2: *"Is it man-made?"* → **"No"**

...7 more questions later...

Player 2 guesses **"Is it an Elephant?"**

→ **Player 2 Wins!**

</div>
</div>

<p style={{ marginTop: 'auto', paddingTop: '1rem', fontSize: '1em', color: 'var(--color-muted)' }}>In Temporal terms, each LLM call runs as an <strong>Activity Task</strong>.</p>

</Slide>

<Slide tab="demo">

During the game, Player 2 will have an advantage. Player 2 will get help from:

# <span style={{ color: 'var(--accent)' }}>Logfire</span> Agent Optimization

<div style={{ marginTop: '1rem', fontSize: '1.1rem', lineHeight: 1.8 }}>

- *Diagnoses before it prescribes.* Classifies each failure — confusing instructions, contradictory goals, ambiguous tools or prompt non-compliance — before drafting anything.
- *An edit, not a re-write.* Your prompt remains yours, it's just tuned to perform better in the real world.
- *Every suggestion is reviewable.* Each change comes with a problem → solution citation and the exact trace IDs that drove it. Approve, refine, or reject — you see the reasoning.
- *Runs on the OTel traces you already send.* No new instrumentation. No separate eval pipeline. Launch sooner and optimize based on real usage, not artificial datasets. 

</div>

</Slide>
