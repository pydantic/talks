# What is "codemode"?

<div style={{ display: 'grid', gridTemplateColumns: '1.3fr 1fr', gap: '2rem', alignItems: 'start' }}>

<div>

- aka "Programatic tool calling", CodeAct, and RLM (Recursive Language Models)
- The model writes Python (or js); the Python calls tools
- Tool composition, branching, looping - all happen in **code**, not round-trips
- Intermediate state stays in the runtime, not in the context window
- The agent only ever sees **two** tools: `run_code` and `find_tools`
- "1000 tools in 1000 tokens" is the promise

</div>

# Introducing Monty

- A **minimal Python interpreter, written in Rust**, for code written by AI
- Safe by default: no filesystem, env, or network unless you grant it
- Pass in your own functions - the model calls them as ordinary Python
- **Type-checks** code (via `ty`) before it runs
- **Snapshottable** - pause at an external call, resume later, anywhere
- Callable from **Rust, Python, or JavaScript**

**Tradeoff:** Monty will not run your existing Python app - it's a Python *subset*.

# Step 1 · Parse to AST

<div style={{ display: 'grid', gridTemplateColumns: '1.3fr 1fr', gap: '2rem', alignItems: 'start' }}>

<div>

- **ruff** parses the source — same AST as Python tooling
- Monty walks the AST eagerly into its own `ParseNode` / `Expr`
- Exit early if we encounter a syntax error, or syntax we don't yet support

</div>

<AstExample />

</div>

</Slide>

<Slide tab="how">

# Step 2 · Resolve names

<div style={{ display: 'grid', gridTemplateColumns: '1.3fr 1fr', gap: '2rem', alignItems: 'start' }}>

<div>

- Second pass classifies every name: **global**, **local**, **free** (closure), or **`LocalUnassigned`**
- String keys become integer `NamespaceId` slot indices
- `LocalUnassigned` = name read but never bound → host-provided external function
- Closure-cell capture decided here so nested functions can reach enclosing locals

</div>

<ResolveExample />

</div>

</Slide>

<Slide tab="how">

# Step 3 · Compile to bytecode

<div style={{ display: 'grid', gridTemplateColumns: '1.3fr 1fr', gap: '2rem', alignItems: 'start' }}>

<div>

- Convert the AST into **bytecode** instructions
- Effectively a `list[int]` - where each `int` is an opcode and its operands
- This compile step is similar to cpython - why we have `.pyc` files

```python
greeting = 'Hello'
who = get_who(greeting)
f'{greeting} {who}'
```

</div>

<div style={{ display: 'grid', gridTemplateColumns: '0.9fr 1.5fr', gap: '2rem', alignItems: 'start' }}>

<div>

- much like other interpreters
- Two stacks: an **operand stack** for values and a **`CallFrame` stack** for activations

# Step 5 · Pause for the host, resume, repeat

<div style={{ display: 'grid', gridTemplateColumns: '1fr 1.5fr', gap: '2rem', alignItems: 'start' }}>

<div>

- Run the code until we hit an external call
- Then return to the host
- Ask it to call the function

</div>

<div>
```python

class Monty:
    def start(self) -> CallTool | Done: ...
    def resume(self, result: ReturnValue | Exception) -> RunProgress: ...

run = Monty(code)
progress = run.start()

while isinstance(progress, CallTool):
    # e.g. CallTool("get_who", ("Hello",))
    # we could call run.dump() here if we wanted to
    result = call_tool(progress.name, progress.args)
    progress = run.resume(result)

print(progress)
```
